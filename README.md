# Task 1: To-Do List App

I made this simple task manager application to practice CRUD (Create, Read, Update, Delete) patterns using Python and Flask.

## Why I built this
I wanted to build a tool that actually works, so I moved beyond just storing data in memory and built this to persist tasks in a real database. It was a great way to learn how to structure back-end logic, handle database connections, and create a custom front-end without relying on heavy frameworks.

## Features
- **Add Tasks:** Quickly add what you need to get done.
- **Set Deadlines:** Keep track of your time with a simple reminder system.
- **Clean List View:** See your tasks clearly.
- **Delete Options:** Easily remove items once you're finished.
- **Data Persistence:** Uses SQLite, so your tasks are saved even after you restart the app.

## Tech Stack
- **Backend:** Python, Flask
- **Database:** SQLite
- **Frontend:** HTML5, CSS3

# Task 2:  Alarm Clock App 

I built this digital alarm clock application to get hands-on experience with full-stack web development. It’s a functional tool that allows users to set, view, and manage alarms in real-time.

## Why I build this?

I wanted to bridge the gap between "tutorial code" and building something that actually works. My main goal was to understand how a web interface communicates with a backend database, handles data persistence, and manages real-time updates—all while fixing the inevitable bugs that pop up when you're learning.

## Features

**Custom Alarms:** Set a specific time and label for each alarm.

**Persistent Storage:** Alarms are saved in a database, so they stay there even after you refresh or restart the server.

**Management:** Easily add new alarms or remove them with a single click.

**Real-time Clock:** A responsive display that keeps you updated with the current time.

# Tech Stack
**Backend:** Python (Flask)
**Database:** SQLite
**Frontend:** HTML5, CSS3, and Jinja2
**Logic:** Vanilla JavaScript
